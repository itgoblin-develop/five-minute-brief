## 1. Executive Summary

본 문서는 **출근길 모바일 뉴스레터 서비스**의 MVP(Minimum Viable Product) 개발을 위한 제품 요구사항을 정의합니다.

### 핵심 가치 제안

바쁜 직장인들이 **출근 시간 5분 이내에 오늘의 주요 뉴스를 카드 형태로 빠르게 소비**할 수 있도록 설계된 모바일 우선 웹 서비스입니다.

**(가칭) 오늘 5분 : 주 5일 출근하는 날, 늘 5분간 볼 수 있는 출근길 모바일 뉴스레터 서비스**

### 주요 특징

- 선택의 피로 없이 큐레이션된 핵심 뉴스만 제공
- 카드 뉴스 형태의 직관적인 스와이프 UI
- 카테고리별(트렌드, 경제, 사회, 재테크) 필터링
- 사용자의 뉴스 열람 패턴을 학습하여 최적의 시간에 푸시 알림 발송
- 비로그인 사용자는 3개 레터 확인 가능, 나머지는 썸네일로만 노출하여 궁금증 유발
- 재구성된 뉴스 콘텐츠로 가독성 높은 뉴스 소비 경험 제공

## 2. 배경 및 목적

### 2.1 배경 (Background)

현대 직장인들이 겪는 뉴스 소비의 문제점:

1. **선택의 피로**: 아침 출근 시간에 어떤 뉴스를 읽어야 할지 결정하기 어려움
2. **시간 부족**: 긴 기사를 읽을 시간이 없지만, 트렌드와 필수 이슈는 놓치고 싶지 않음
3. **정보 과부하**: 기존 뉴스 앱은 너무 많은 정보를 제공해 핵심만 빠르게 파악하기 어려움
4. **맞춤형 부재**: 개인화된 뉴스 큐레이션 서비스는 있지만, 출퇴근 시간대에 특화된 서비스는 부족

### 2.2 목적 (Objectives)

1. 출근 시간 10분 이내에 소비 가능한 요약 뉴스를 카드 형태로 제공하는 모바일 뉴스레터 MVP 개발
2. 카테고리별(트렌드, 경제, 사회, 재테크) 뉴스 필터링 기능 구현
3. 사용자의 뉴스 열람 패턴을 분석하여 최적의 시간에 푸시 알림 발송 (알림 허용 시)
4. 프로덕션 배포까지 완료하여 실사용자 피드백 수집

### 2.3 성공 기준 (Success Criteria)

### 기능 기준

- 핵심 플로우 4개(메인 피드, 카테고리 탭, 로그인, 상세보기)가 오류 없이 동작
- 뉴스 API 연동이 정상 작동하여 실시간 뉴스 제공

### 품질 기준

- 페이지 로딩 시간 p95 **3초 이내**
- 푸시 알림 발송 성공률 **95% 이상**
- 모바일 반응형 지원 (iOS 14+, Android 10+)

### 산출 기준

- 프로덕션 배포 URL 제공
- 사용자 가이드 1페이지 작성
- API 문서화 완료

### 사용자 지표 (출시 후 2주)

- **DAU 100명 이상**
- **평균 세션 시간 5분 이상**
- **뉴스 카드 평균 3개 이상 읽기**

## 4. 핵심 기능 (Core Features)

### 4.1 MVP 포함 기능 ✅

### 1. 메인 뉴스 피드

- 카드 형태의 스와이프 UI로 뉴스 제공
- 제목, 2~3줄 요약, 이미지, 날짜 표시
- 좌우 스와이프로 다음 뉴스 이동

### 2. 카테고리별 필터링

- 상단 4개 탭: 트렌드, 경제, 사회, 재테크
- 탭 클릭 시 해당 카테고리 뉴스만 표시

### 3. 요약 뉴스 제공

- 제목 + 2~3줄 요약 (200자 이내)
- 썸네일 이미지 (뉴스에 재구성된 내용에 새로 제작된 이미지)
- 상세보기 클릭 시 재구성된 뉴스 콘텐츠 제공 (원문 페이지 이동 아님)
- 뉴스레터 목록으로 돌아가기 버튼 제공
- 뉴스 데이터 수집 시간 기본값 설정
- 뉴스 전환 시간 기준 (DAY)

### 4. 재구성된 뉴스 콘텐츠

- 원문을 읽기 쉽게 재구성한 콘텐츠
- 단락 구분, 핵심 내용 강조 등으로 가독성 향상
- 뉴스레터 목록 페이지로 쉽게 돌아갈 수 있는 네비게이션

### 5. 로그인/회원가입

- 이메일 기반 인증 쉬운 회원가입 구조
- **JWT (JSON Web Token) 토큰 발급 및 세션 관리**
  - JWT란? 사용자 인증 정보를 안전하게 전송하기 위한 토큰 기반 인증 방식
  - 로그인 성공 시 서버가 암호화된 토큰을 생성하여 사용자에게 전달
  - 이후 모든 요청에 이 토큰을 포함하여 사용자 신원 확인
  - 세션 관리: 토큰 유효 기간(24시간) 동안 자동 로그인 유지
  - 보안: 서버 측 비밀키로 암호화되어 위변조 방지

### 6. 비로그인 제한

- **3개 레터까지 전체 내용 확인 가능**
- 4번째 레터부터는 **썸네일(제목 + 이미지 + 짧은 티저)만 노출**하여 궁금증 유발
- 썸네일 클릭 시 로그인 유도 팝업 표시
- "더 많은 뉴스를 보려면 로그인하세요" 메시지

### 7. 푸시 알림 (스마트 타이밍)

- **알림 허용 시에만 작동**
- **사용자의 뉴스 열람 패턴 자동 분석**
  - 어느 시간대에 뉴스를 가장 많이 보는지 학습
  - 3일 이상의 패턴 데이터 수집 후 최적 시간 자동 설정
  - 예: 주로 오전 8시에 접속하면 → 오전 8시에 알림 발송
- 수동 시간 설정 가능 (사용자가 직접 원하는 시간 지정)
- 알림 on/off 토글
- FCM (Firebase Cloud Messaging) 기반

### 8. 설정 화면

- **알림 설정**
  - 스마트 알림: AI가 자동으로 최적 시간 결정 (기본값)
  - 수동 알림: 사용자가 직접 시간 설정
  - 알림 on/off 스위치
- 프로필 정보 수정 (닉네임, 비밀번호)

### 9. 상세 뉴스 화면

- 카드 클릭 시 재구성된 뉴스 콘텐츠 페이지로 이동
- 읽기 쉽게 재구성된 뉴스 본문
- 전체 뉴스 모아보기
- 카드형/리스트형 선택 토글
- **댓글/커뮤니티 기능**
- **북마크/저장 기능**

### 4.2 MVP 제외 기능 (v2 이후) ❌

- AI 기반 개인화 추천 (사용자 행동 학습)
- 음성 읽기 기능 (TTS)
- 오프라인 저장/캐싱
- 다국어 지원 (영어/중국어)
- 소셜 로그인 (카카오, 구글 등)
- 뉴스 검색 기능
- 실시간 알림 (속보)

## 5. 사용자 스토리 (User Stories)

### 주요 사용자 스토리

**US-01**: 사용자로서 아침 출근 시간에 푸시 알림을 받아 오늘의 주요 뉴스를 빠르게 확인하고 싶다.

**US-02**: 사용자로서 관심 있는 카테고리(경제, 재테크 등)를 선택해 해당 뉴스만 모아보고 싶다.

**US-03**: 사용자로서 짧은 요약으로 뉴스의 핵심을 파악하고, 관심 있으면 재구성된 뉴스 콘텐츠를 읽고 싶다.

**US-04**: 사용자로서 스와이프 제스처로 다음 뉴스를 빠르게 넘기며 여러 뉴스를 효율적으로 소비하고 싶다.

**US-05**: 사용자로서 내가 뉴스를 자주 보는 시간대에 자동으로 알림을 받고 싶다 (또는 수동으로 시간 설정).

**US-06**: 비로그인 사용자로서 3개 레터를 체험해보고, 나머지는 썸네일로 미리보기하여 서비스 가치를 확인한 후 가입 여부를 결정하고 싶다.

**US-07**: 운영자로서 뉴스 API 오류나 시스템 장애를 빠르게 감지하고 대응하고 싶다.

**US-08**: 사용자로서 재구성된 뉴스를 읽은 후, 뉴스레터 목록으로 쉽게 돌아가서 다음 뉴스를 계속 볼 수 있기를 원한다.

## 8. 데이터 정책 및 리스크 관리

### 8.1 데이터 수집 및 저장

### 수집 방식

- 뉴스 API (네이버 뉴스, SNS 인스타그램, 유투브)
- 사용자 입력 (회원 정보)
- 카테고리별 최대 20개 (일)

### 저장 데이터

- **뉴스 원문**: 저장하지 않음 (API에서 실시간 조회)
- **뉴스 메타데이터**: 제목, 요약, 날짜 저장
- **사용자 정보**: 이메일(암호화), 해싱된 비밀번호, 디바이스 토큰
- **알림 설정**: 알림 시간, 푸시 on/off 상태

### 보관 기간

- **사용자 데이터**: 회원 탈퇴 시 즉시 삭제
- **로그 데이터**: 30일

### 금지 데이터

- 주민번호, 계좌번호, 카드번호 등 민감정보
- 뉴스 원문 전체 복사 (저작권 침해 방지)
- 사용자 행동 추적 데이터 (위치, 브라우징 기록 등)

### 8.2 법적 리스크 및 대응 전략

| 리스크            | 내용                   | 회피 전략    |
| ----------------- | ---------------------- | ------------ |
| **개인정보 유출** | 사용자 정보 해킹, 유출 | • HTTPS 적용 |

• 비밀번호 해싱 (bcrypt)
• 최소 정보만 수집
• 개인정보 처리방침 게시 |
| **API 과다 호출** | API 제공사 제재 | • 캐싱 적용 (1시간)
• Rate limiting
• API 호출 최적화 |
| **푸시 알림 스팸** | 과도한 알림으로 사용자 이탈 | • 사용자 설정 시간만 발송
• On/off 토글 제공
• 하루 1회 제한 |

### 8.3 개인정보 처리방침 필수 항목

1. 수집하는 개인정보 항목 (이메일, 닉네임, 디바이스 토큰)

5. 개인정보 처리 위탁 여부 (FCM 서버)
6. 이용자 권리 및 행사 방법 (열람, 정정, 삭제, 탈퇴)

## 10. 사용자 플로우 (User Flows)

### 10.1 비로그인 사용자 플로우

```
1) 앱/웹 접속
   ↓
2) 메인 피드 (카드 뉴스 3개까지 열람)
   ↓
3) 4번째 카드 접근 시 → 로그인 유도 팝업
   ↓
4-a) 로그인 → 전체 콘텐츠 이용
4-b) 회원가입 → 로그인 → 전체 콘텐츠 이용
4-c) 닫기 → 3개 카드만 계속 이용

```

### 10.2 로그인 사용자 플로우

```
1) 로그인
   ↓
2) 메인 피드 (전체 뉴스 무제한 열람)
   ↓
3-a) 카테고리 탭 클릭 → 해당 카테고리 뉴스 목록
3-b) 카드 클릭 → 상세 화면 → 원문 링크 이동
3-c) 설정 버튼 → 설정 화면 (알림 시간, 푸시 on/off)
   ↓
4) 로그아웃

```

### 10.3 푸시 알림 플로우

```
1) 사용자: 설정에서 알림 시간 설정 (예: 오전 8시)
   ↓
2) 서버: 매일 오전 8시에 FCM 푸시 발송
   ↓
3) 사용자: 알림 클릭 → 앱/웹 메인 피드 이동

```

## 16. 부록 (Appendix)

### 16.1 용어 정의

- **카드 뉴스**: 뉴스 제목, 요약, 이미지를 카드 형태로 표시한 UI 컴포넌트
- **스와이프**: 좌우로 넘기며 다음 뉴스를 확인하는 제스처 기반 인터랙션
- **카테고리**: 뉴스를 분류하는 주제 단위 (트렌드, 경제, 사회, 재테크)
- **푸시 알림**: 설정한 시간에 사용자 디바이스로 전송되는 알림 메시지
- **비로그인 사용자**: 회원가입/로그인하지 않은 사용자 (기능 제한)
- **원문 링크**: 뉴스의 전체 내용을 볼 수 있는 언론사 URL
- **MVP**: 이번 프로젝트에서 반드시 구현해야 하는 최소 기능
- **FCM**: Firebase Cloud Messaging (푸시 알림 서비스)
