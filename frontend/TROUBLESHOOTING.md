# 프론트엔드 API 호출 문제 해결 가이드

## 🔍 400 에러 발생 시 확인 사항

### 1. Network 탭에서 응답 확인

1. **개발자 도구 열기** (F12)
2. **Network 탭** 클릭
3. **signup 요청 클릭**
4. **Response 탭** 확인

**예상 응답:**
```json
{
  "success": false,
  "error": "이미 사용 중인 이메일입니다"
}
```

### 2. 가능한 원인

#### 원인 1: 이메일 중복
- **에러 메시지**: "이미 사용 중인 이메일입니다"
- **해결**: 다른 이메일 사용

#### 원인 2: 닉네임 중복
- **에러 메시지**: "이미 사용 중인 닉네임입니다"
- **해결**: 다른 닉네임 사용

#### 원인 3: 이메일 형식 오류
- **에러 메시지**: "올바른 이메일 형식이 아닙니다"
- **해결**: 올바른 이메일 형식 사용 (예: `test@example.com`)

#### 원인 4: 닉네임 길이 오류
- **에러 메시지**: "닉네임은 2자 이상 20자 이하여야 합니다"
- **해결**: 2-20자 사이의 닉네임 사용

#### 원인 5: 비밀번호 길이 오류
- **에러 메시지**: "비밀번호는 최소 8자 이상이어야 합니다"
- **해결**: 8자 이상의 비밀번호 사용

### 3. 해결 방법

#### 방법 1: Network 탭에서 응답 확인
1. Network 탭에서 `signup` 요청 클릭
2. Response 탭에서 에러 메시지 확인
3. 에러 메시지에 따라 수정

#### 방법 2: 콘솔에서 확인
1. Console 탭 열기
2. 에러 메시지 확인
3. 상세 정보 확인

#### 방법 3: 코드 수정
- 이메일 중복 방지: 타임스탬프 추가 (이미 적용됨)
- 에러 메시지 표시 개선 (이미 적용됨)

## 🧪 테스트 방법

### 1. 회원가입 테스트

**성공 케이스:**
- 이메일: `test123456@frontend.com` (고유한 이메일)
- 닉네임: `테스트유저` (2-20자)
- 비밀번호: `test1234` (8자 이상)

**실패 케이스:**
- 이미 존재하는 이메일
- 이미 존재하는 닉네임
- 이메일 형식 오류
- 닉네임 길이 오류
- 비밀번호 길이 오류

### 2. 로그인 테스트

**성공 케이스:**
- 회원가입한 이메일과 비밀번호 사용

**실패 케이스:**
- 존재하지 않는 이메일
- 잘못된 비밀번호
